📋
产品简介

《百宝箱》是一款专为 Windows 用户打造的本地小工具集合，聚焦办公常用场景，无需注册账号、无需联网，全部处理在本地完成，数据安全可靠。

🖼️

图片批量压缩

支持拖拽批量导入，按目标文件大小（KB）或质量（%）两种模式精准压缩，专为政府网站、表单系统等对文件大小有严格要求的场景设计。

📄

PDF 转 Word

将 PDF 文档转换为可编辑的 Word（.docx）格式。支持数字型 PDF（保留排版）和扫描版 PDF（OCR 识别文字），可选择是否保留图片和格式。

🎬

屏幕录制

调用 Windows 系统 GDI 能力，实时截帧并通过 FFmpeg 编码为高清 MP4 视频。支持帧率调节、区域录制（规划中）、暂停/继续。

🎨

5 套主题色

内置海天蓝、青翠绿、星空紫、暖阳橙、玫瑰红五套主题，设置页面一键切换，即时生效，个性化你的工作界面。

ℹ️
本地优先原则

百宝箱所有核心功能均在本地计算机执行，不向任何服务器上传您的文件内容。广告位功能（如已启用）仅拉取广告素材，不涉及用户文件。

安装指南

💡
两种安装方式

方式 A（推荐）：使用打包好的 BaibaoBOX-v1.0.1-Setup.exe 安装包，双击安装，无需配置 Python；
方式 B（开发者）：克隆源代码，手动配置 Python 环境后运行，适合开发调试。

方式 A — 安装包安装（推荐普通用户）

  • 1

    下载安装包

    从官方渠道获取 BaibaoBOX-v1.0.1-Setup.exe（约 100MB，含 Python 运行时 + FFmpeg）。

  • 2

    以管理员身份运行安装程序

    右键 → "以管理员身份运行"。若系统弹出 UAC 提示，点击「是」允许。

  • 3

    选择安装目录

    默认路径：C:\Program Files\BaibaoBOX\。建议保持默认，不要安装到中文路径或含空格的目录。

  • 4

    完成安装

    安装程序将自动创建桌面快捷方式和开始菜单项。点击"完成"即可。

  • 5

    启动验证

    双击桌面图标，主界面正常加载则安装成功。首次启动会在 桌面\百宝箱输出\ 自动创建默认输出文件夹。

方式 B — 源码运行（开发者/高级用户）

第一步：安装 Python 3.11 ~ 3.13

访问 python.org 下载并安装 Python，安装时务必勾选 「Add Python to PATH」。

# 验证 Python 安装
python --version
# 应输出: Python 3.13.x 或 Python 3.11.x

pip --version
# 应输出: pip XX.X.Xcmd

第二步：获取项目代码

# 克隆仓库（或直接下载 ZIP 解压）
git clone https://github.com/your-org/baibaoxiang.git
cd baibaoxiangcmd

第三步：创建虚拟环境（推荐）

# 创建 .venv 虚拟环境（推荐，避免污染系统 Python）
python -m venv .venv

# 激活虚拟环境
.venv\Scripts\activate

# 确认激活成功（命令行前缀应显示 (.venv)）
python --versioncmd

第四步：安装 Python 依赖

# 一键安装所有依赖（约 300MB，首次需要等待几分钟）
pip install -r requirements.txt

# 如果下载速度慢，可使用国内镜像源
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simplecmd

第五步：安装 FFmpeg（屏幕录制必需）

推荐方式：winget 一键安装（Windows 10/11）

# 使用 winget（系统内置包管理器）
winget install --id Gyan.FFmpeg

# 验证安装
ffmpeg -version
# 应输出: ffmpeg version 6.x.x ...cmd

手动安装方式：

  1. 访问 ffmpeg.org 下载 Windows 版本（选择 gyan.dev 的 full build）
  2. 解压到如 C:\ffmpeg\（路径不要含中文）
  3. 将 C:\ffmpeg\bin 添加到系统环境变量 PATH
  4. 打开新的 cmd 窗口，运行 ffmpeg -version 验证
⚠️
FFmpeg 路径不要含中文

FFmpeg 的安装路径中不得包含中文字符或空格，否则会导致录制失败。如果系统 PATH 无法识别，可在百宝箱「设置 → 录制设置 → FFmpeg 路径」中手动填写完整路径（如 C:\ffmpeg\bin\ffmpeg.exe）。

第六步：运行应用

# 确保虚拟环境已激活
python main.pycmd

图片批量压缩

支持格式

格式压缩模式说明
JPG / JPEG按大小 & 按质量最常用格式，支持精确大小控制
PNG按大小（无损压缩）PNG 使用无损压缩，按大小模式会转为 JPEG 以节省空间；按质量模式不压有损
WebP按大小 & 按质量现代格式，压缩率更高
BMP按大小自动转换为 JPEG 压缩
TIFF按大小使用 LZW 无损压缩

压缩模式说明

📐 按大小压缩（Size Mode）

设置目标文件大小（KB），程序使用二分搜索算法自动找到最接近的压缩参数，反复迭代直到结果大小落在目标 ±20KB 范围内。

  • 适用场景：政府网站、表单系统要求"图片不超过 500KB"等明确大小限制场景
  • 默认目标：500 KB
  • 可在设置页面修改默认值

🎚️ 按质量压缩（Quality Mode）

直接指定压缩质量（1~100），压缩后大小以实际为准，不保证特定大小。

  • 适用场景：批量处理图片、控制总体质量水平
  • 默认质量：75（画质与大小的较好平衡点）
  • 建议范围：60~85 可兼顾效果与大小

操作步骤

  • 1

    点击侧边栏「图片压缩」

    进入图片压缩功能页。

  • 2

    导入图片

    方式一：将图片文件拖入中间的拖拽区（支持同时拖入多张）；
    方式二：点击拖拽区选择文件，弹出文件选择对话框（可多选）。
    图片将显示在下方文件列表中，含文件名、原始大小、状态信息。

  • 3

    选择压缩模式并设置参数

    点击「按大小」或「按质量」切换模式，在右侧输入目标大小（KB）或质量（%）。

  • 4

    选择输出目录（可选）

    默认输出到「桌面 → 百宝箱输出」文件夹。点击「更改输出目录」可自定义。

  • 5

    开始压缩

    点击「开始压缩」按钮。列表中每个文件会显示进度，压缩完成后显示原始大小 → 压缩后大小 → 节省比例。

  • 6

    查看结果

    点击「打开输出目录」可直接打开结果文件夹。压缩文件命名格式：原文件名_compressed.jpg。

⚠️
PNG 格式特别说明

PNG 是无损格式，「按质量」模式对 PNG 文件不会产生明显的有损压缩效果。若需要大幅缩小 PNG 文件，请选择「按大小」模式，程序会自动转为 JPEG 格式以实现减小文件大小的目的。原始 PNG 文件不会被修改。

PDF 转 Word

PDF 文档类型识别

PDF 类型特征转换方式效果
数字型 PDF 可选中、可复制文字 pdf2docx 直接解析 ⭐⭐⭐⭐⭐ 最佳，保留排版
扫描版 PDF 图片扫描，无法选中文字 PyMuPDF 渲染 + Tesseract OCR ⭐⭐⭐ 依赖 OCR 精度
混合型 PDF 部分页数字、部分扫描 自动分析每页类型分别处理 ⭐⭐⭐⭐ 较好

操作步骤

  • 1

    进入「PDF 转 Word」页面

    点击侧边栏 PDF 转 Word 入口。

  • 2

    导入 PDF 文件

    拖拽 PDF 文件到拖拽区，或点击选择文件（支持批量导入多个 PDF）。

  • 3

    配置转换选项

    ☑️ 保留排版格式：尽量保留原 PDF 的字体、间距、对齐方式（数字型 PDF 效果更好）
    ☑️ 保留图片：将 PDF 中的图片嵌入到 Word 文档中
    下拉选择文档类型：普通 / 扫描版 / 自动识别

  • 4

    开始转换

    点击「开始转换」，进度条逐文件推进。大文件（100页以上）可能需要 1~2 分钟。

  • 5

    获取结果

    转换完成后，结果 .docx 文件保存在输出目录，文件名与原 PDF 同名。

💡
扫描版 PDF 转换质量取决于图像清晰度

扫描 OCR 的识别精度与原始 PDF 的图像分辨率成正比。建议扫描时使用 ≥300 DPI，黑白/灰度文档效果优于彩色。手写内容目前不支持 OCR 识别。

⚠️
扫描版 PDF 需额外安装 Tesseract OCR

处理扫描版 PDF 需要本地安装 Tesseract OCR 引擎。安装地址：github.com/tesseract-ocr/tesseract，需同时下载中文语言包 chi_sim.traineddata。数字型 PDF 无此要求。

屏幕录制

录制参数说明

参数默认值可选范围说明
帧率 (FPS)155 / 10 / 15 / 20 / 25 / 30越高画面越流畅，但文件越大、CPU 占用越高
编码器libx264libx264 / libx265libx264 兼容性最好，libx265 文件更小但编码更慢
输出格式MP4MP4当前版本固定输出 MP4（H.264）
录制区域全屏全屏 / 自定区域（规划中）v1.0 仅支持全屏录制

录制前准备

  • 确认 FFmpeg 已正确安装（参见「安装指南」第五步）
  • 确认输出目录有足够磁盘空间（1080P 15fps 大约每分钟 30~80MB）
  • 关闭其他高负载程序，避免录制帧率不稳定

操作步骤

  • 1

    进入「屏幕录制」页面

    点击侧边栏屏幕录制入口。

  • 2

    设置录制参数

    在控制台上方选择帧率和编码器；选择输出目录（默认桌面百宝箱输出）。

  • 3

    开始录制

    点击红色「开始录制」按钮，计时器开始计时，按钮变为「暂停」和「停止」。此时切换到其他窗口，录制不会中断。

  • 4

    暂停 / 继续

    点击「暂停」按钮暂停录制，计时器停止；点击「继续」恢复。

  • 5

    停止录制

    点击「停止」按钮，程序等待 FFmpeg 完成最后帧的编码（约 1~3 秒），然后显示输出文件路径。文件命名格式：录屏_YYYYMMDD_HHMMSS.mp4。

  • 6

    查看录制文件

    点击「打开输出目录」，用默认播放器（如 Windows 媒体播放器）预览录制结果。

🔴
直接关闭窗口会自动安全停止录制

v1.0.1 已修复关窗口时录制未停止的问题。现在关闭主窗口时，程序会自动等待 FFmpeg 完成编码并安全退出，不会产生损坏的视频文件。

推荐开发工具链

工具推荐版本用途

完整开发环境搭建命令

# 1. 克隆项目
git clone https://github.com/your-org/baibaoxiang.git
cd baibaoxiang

# 2. 创建虚拟环境
python -m venv .venv
.venv\Scripts\activate

# 3. 安装所有依赖（含开发工具）
pip install -r requirements.txt

# 4. 安装 pywin32 后处理脚本（必须执行一次）
python .venv\Scripts\pywin32_postinstall.py -install

# 5. 验证关键依赖
python -c "from PyQt6.QtWidgets import QApplication; print('PyQt6 OK')"
python -c "from PIL import Image; print('Pillow OK')"
python -c "import fitz; print('PyMuPDF OK')"
python -c "import pdf2docx; print('pdf2docx OK')"
python -c "import win32gui; print('pywin32 OK')"

# 6. 运行应用
python main.pycmd
⚠️
pywin32 安装后需执行 postinstall

pywin32 安装后必须运行 pywin32_postinstall.py -install 才能正确注册 COM 组件，否则 win32gui 会报 ImportError。这是 pywin32 的官方要求，每次创建新虚拟环境后都要执行一次。

baibaoxiang/ # 项目根目录 ├── main.py # 应用入口，初始化 QApplication ├── requirements.txt # Python 依赖声明 ├── build.py # PyInstaller 打包脚本 ├── PROJECT.md # 项目说明文档 │ ├── src/ # 全部源代码 │ ├── config.py # 全局配置管理（JSON 持久化，单例） │ ├── database.py # SQLite 数据访问层（历史记录+广告缓存） │ ├── signals.py # Qt 全局信号总线（单例） │ ├── worker_threads.py # 后台线程：压缩/转换/录制 │ ├── main_window.py # 主窗口（侧边栏+页面堆栈+主题加载） │ │ │ ├── modules/ # 核心功能模块（纯 Python，不含 Qt） │ │ ├── image_compressor.py # Pillow 压缩，二分搜索算法 │ │ ├── pdf_converter.py # pdf2docx 转换器 │ │ ├── screen_recorder.py # pywin32 GDI + FFmpeg 管道录制 │ │ └── ad_manager.py # 远程广告拉取+缓存 │ │ │ ├── pages/ # UI 页面（QWidget 子类） │ │ ├── home_page.py # 首页：统计+快捷入口+历史记录 │ │ ├── compress_page.py # 图片压缩页面 │ │ ├── pdf_word_page.py # PDF 转 Word 页面 │ │ ├── recorder_page.py # 屏幕录制页面 │ │ ├── guide_page.py # 使用指南三标签页 │ │ └── settings_page.py # 系统设置页面 │ │ │ └── theme/ # 主题样式表 │ ├── blue.qss # 海天蓝（默认） │ ├── teal.qss # 青翠绿 │ ├── purple.qss # 星空紫 │ ├── amber.qss # 暖阳橙 │ └── rose.qss # 玫瑰红 │ ├── data/ # 运行时数据（自动生成，勿手动删除） │ ├── baibaobox.db # SQLite 数据库（历史记录） │ ├── config.json # 用户配置文件 │ ├── logs/ # 运行日志 │ └── cache/ # 广告素材缓存 │ ├── docs/ # 文档 │ ├── user-manual.html # 本使用说明书 │ ├── tech-spec.html # 技术方案文档 │ └── bug-report.html # Bug 报告（历史归档） │ ├── prototype/ # 产品原型 │ └── index.html # 交互式 HTML 原型 │ └── build/ # 打包输出目录（自动生成）

架构设计原则

  • 前后端分离：modules/ 中的功能模块不依赖任何 Qt 类，可独立测试
  • 信号总线解耦：页面与模块之间全部通过 SignalBus 通信，避免直接依赖
  • 配置单例：全局只有一个 AppConfig 实例，所有页面共享，保证一致性
  • 后台线程：所有耗时操作（压缩/转换/录制）均在 QThread 中执行，UI 不卡顿

打包前准备

确保 PyInstaller 已安装（pip install pyinstaller），且当前系统上所有功能均可正常运行。打包需要在 Windows 系统上进行，macOS/Linux 无法交叉编译 Windows 应用。

方式一：一键打包脚本（推荐）

# 激活虚拟环境
.venv\Scripts\activate

# 运行打包脚本（自动处理所有参数）
python build.pycmd

打包完成后，可执行文件位于 dist/BaibaoBOX.exe，运行时大约 80~120MB。

方式二：手动 PyInstaller 命令

pyinstaller --onefile --windowed \
  --name=BaibaoBOX \
  --add-data "src/theme;src/theme" \
  --add-data "src/resources;src/resources" \
  --hidden-import=PyQt6.sip \
  --hidden-import=win32api \
  --hidden-import=win32gui \
  --hidden-import=win32con \
  main.pycmd

广告位配置

百宝箱在每个功能页面顶部预留了广告横幅位，通过远程 API 拉取广告素材，完全解耦于核心功能，失败静默降级（不显示广告但不影响任何功能）。

广告位设计原则

  • 独立线程拉取，不阻塞 UI 线程
  • 本地 SQLite 缓存最近一次广告，即使 API 不可达仍可展示缓存内容
  • 网络故障时静默失败，用户不感知错误
  • 可随时在设置页开关广告功能

广告 API 接口规范

广告服务端需实现以下 HTTP 接口：

GET /api/ads
Authorization: Bearer <your_api_key>
Content-Type: application/json

// 期望响应格式：
{
  "ads": [
    {
      "title":     "广告标题文字",
      "image_url": "https://cdn.example.com/ad.png",
      "link_url":  "https://www.example.com/landing"
    }
  ]
}JSON

配置步骤

  • 1

    搭建广告 API 服务

    按以上接口规范实现服务端（可使用 Flask、FastAPI、Node.js 等任意框架）。

  • 2

    在设置页填写配置

    打开百宝箱 → 系统设置 → 广告配置；填写 API 地址、密钥，设置刷新间隔（分钟），开启「启用广告位」开关。

  • 3

    保存并验证

    点击「保存设置」，重启应用后广告位将在功能页顶部展示。

常见问题 (FAQ)

安装完依赖后运行 main.py，提示 ModuleNotFoundError: No module named 'PyQt6'
通常是虚拟环境未激活。请先运行 .venv\Scripts\activate（命令行前缀显示 (.venv)），再重新运行 python main.py。若激活后仍报错，执行 pip install PyQt6 重新安装。
点击「开始录制」后提示「未找到 FFmpeg，请先安装」
FFmpeg 未安装或未加入系统 PATH。解决方案：
① 运行 winget install Gyan.FFmpeg（需重启终端使 PATH 生效）；
② 或手动安装后，在百宝箱「设置 → 录制设置 → FFmpeg 路径」中填写完整路径，如 C:\ffmpeg\bin\ffmpeg.exe。
图片压缩后大小比目标值大/小很多，不准确
「按大小」模式使用二分搜索算法，设计精度为目标值 ±20KB。对于非常小的图片（原始文件小于目标值），程序会提示"无需压缩"。对于 PNG 格式，建议选择「按大小」模式，程序会自动转为 JPEG 实现显著压缩。
PDF 转 Word 后文字乱码或版式错乱
版式错乱是 PDF 转 Word 的固有局限性（PDF 是固定排版格式，Word 是流式排版）。建议：
① 确认 PDF 是数字型（可选中文字），如果是扫描版请选择对应选项；
② 对于复杂排版（多栏、表格嵌套），转换后可能需要手动调整；
③ 简单文字类文档转换效果最佳。
扫描版 PDF 转换时提示「Tesseract 未找到」
扫描版 PDF 需要 Tesseract OCR。安装步骤：
① 下载 Tesseract Windows 安装包；
② 安装时勾选中文语言包（Chinese Simplified）；
③ 将 Tesseract 安装目录加入系统 PATH；
④ 运行 tesseract --version 验证。
录制视频播放时卡顿/帧率低
录制帧率低通常是因为 CPU 负载过高。建议：
① 关闭不必要的后台程序；
② 将帧率从 25 降至 15 FPS；
③ 使用 libx264 而不是 libx265（libx265 编码更耗 CPU）；
④ 确保 Python 版本为 3.11+ 且使用 64 位版本。
pywin32 安装后提示 ImportError: DLL load failed
pywin32 安装后需要运行后处理脚本注册 DLL。执行：
python .venv\Scripts\pywin32_postinstall.py -install
若提示权限不足，请以管理员身份运行命令提示符，再重新执行。
主题切换后部分组件颜色没有更新
v1.0.1 已修复主题不同步问题。若仍出现，尝试：关闭当前页面并重新切换到该页面，或重启应用。如果是自定义组件，可能需要手动调用 style().polish(widget)。
data/config.json 误删后如何恢复默认设置
直接重启应用即可。程序检测到配置文件不存在时会自动创建默认配置，所有参数恢复为出厂默认值（蓝色主题、500KB 压缩目标、15FPS 录制等）。历史记录数据库（baibaobox.db）不受影响。
我的文件会被上传到网络吗？
不会。百宝箱所有核心功能（图片压缩、PDF转换、屏幕录制）全部在本地执行，不向任何服务器发送您的文件内容。广告位功能（如启用）仅从服务器拉取广告素材（图片URL和链接），不涉及您的任何文件或操作数据。
