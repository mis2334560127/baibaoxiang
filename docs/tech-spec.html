01 项目概述

百宝箱是一款面向普通 Windows 用户的本地化效率小工具，聚焦三个高频刚需：图片批量压缩、PDF 文档转换、屏幕录制，全程不依赖网络，保护用户数据隐私。同时预留广告位接口，支持后期通过远程数据库动态投放广告变现。

目标用户
需要向政府/企业平台上传材料的行政人员、中小企业文员、日常办公用户
核心价值
零学习成本、全本地处理、一键操作，解决"图片太大上传失败"等痛点
运行环境
Windows 10 / 11（64位），RAM ≥ 4GB，磁盘 ≥ 500MB，无需联网

功能需求矩阵

功能模块核心能力底层依赖优先级
图片批量压缩按大小/质量压缩、批量处理、多格式支持PillowP0

整体采用 单进程多模块 架构，主进程承载 PyQt6 GUI，各功能模块作为独立子模块通过接口调用。广告系统作为独立的网络层，与主功能完全解耦，不影响核心功能的离线使用。

03 技术栈选型

3.1 开发语言

🐍
Python 3.11
主力语言

全程使用 Python 开发。生态丰富，图像处理（Pillow）、PDF处理（pdf2docx）、系统调用等库成熟稳定。PyInstaller 可直接打包为 .exe，用户无需安装 Python 环境。

🗄️
SQL（SQLite）
数据存储

用于历史记录持久化和广告配置缓存。SQLite 是单文件数据库，无需安装，Python 内置支持，适合本地桌面应用。

📋
JSON
配置存储

用户偏好设置（主题色、默认路径、开关状态、广告API配置）以 JSON 格式存储于本地 AppData 目录，人类可读，方便调试。

04 图片批量压缩模块

实现原理

使用 Pillow 库进行图片读取与重编码。压缩本质是在保持视觉效果的前提下降低编码质量（JPEG quality 参数）或缩减分辨率，直到目标文件大小满足要求。批量任务通过 QThreadPool 多线程并发处理，避免阻塞 UI。

按目标大小压缩算法

采用二分搜索法逼近目标大小，在 quality=1~95 区间内迭代，最多 10 次迭代，保证不过度损失画质：

from PIL import Image
import io

def compress_to_size(input_path: str, target_kb: int, output_path: str) -> dict:
    """按目标大小压缩图片，使用二分搜索法逼近"""
    img = Image.open(input_path).convert("RGB")
    
    lo, hi, best_quality = 1, 95, 85
    target_bytes = target_kb * 1024
    
    for _ in range(10):  # 最多迭代10次
        mid = (lo + hi) // 2
        buf = io.BytesIO()
        img.save(buf, format="JPEG", quality=mid, optimize=True)
        size = buf.tell()
        
        if size <= target_bytes:
            best_quality = mid
            lo = mid + 1   # 尝试更高质量
        else:
            hi = mid - 1   # 降低质量
    
    # 用最佳质量写出文件
    img.save(output_path, quality=best_quality, optimize=True)
    original_size = os.path.getsize(input_path)
    final_size = os.path.getsize(output_path)
    
    return {
        "saved_bytes": original_size - final_size,
        "ratio": round((1 - final_size / original_size) * 100, 1),
        "quality": best_quality
    }

多线程批量处理

from PyQt6.QtCore import QRunnable, QThreadPool, pyqtSignal, QObject

class CompressWorker(QRunnable):
    def __init__(self, task: dict, signals):
        super().__init__()
        self.task = task
        self.signals = signals
    
    def run(self):
        try:
            result = compress_to_size(
                self.task["input"], self.task["target_kb"], self.task["output"]
            )
            self.signals.finished.emit(self.task["id"], result)
        except Exception as e:
            self.signals.error.emit(self.task["id"], str(e))

# 使用线程池，最大并发 4 个
pool = QThreadPool.globalInstance()
pool.setMaxThreadCount(4)
for task in tasks:
    pool.start(CompressWorker(task, signals))

05 PDF 转 Word 模块

两种转换路径

路径 A：数字型 PDF（推荐）

PDF 文字可复制，直接使用 pdf2docx 库解析，还原段落、表格、字体样式，转换质量高，速度快（1页约0.5s）。

路径 B：扫描版 PDF（OCR）

先用 pdf2image 将每页转为图片，再调用 Tesseract OCR 识别文字，最后组装为 Word 文档。识别准确率约85~92%（取决于扫描清晰度）。

数字型 PDF 转换核心代码

from pdf2docx import Converter

def convert_pdf_to_docx(pdf_path: str, docx_path: str, callback=None):
    """转换数字型 PDF，支持进度回调"""
    cv = Converter(pdf_path)
    
    def progress_hook(page_idx, total):
        if callback:
            callback(int(page_idx / total * 100))
    
    cv.convert(
        docx_path,
        start=0,
        end=None,
        # 保留格式选项
        parse_lattice_table=True,  # 解析有边框的表格
        parse_stream_table=True,   # 解析无边框的表格
    )
    cv.close()
    return docx_path

扫描版 PDF（OCR）转换流程

from pdf2image import convert_from_path
import pytesseract
from docx import Document

def ocr_pdf_to_docx(pdf_path: str, docx_path: str, lang: str = "chi_sim+eng"):
    """扫描版 PDF OCR 转换"""
    # 第1步：PDF 每页转图片（200dpi 平衡速度和精度）
    images = convert_from_path(pdf_path, dpi=200)
    
    doc = Document()
    for i, img in enumerate(images):
        # 第2步：OCR 识别，同时支持中文和英文
        text = pytesseract.image_to_string(img, lang=lang)
        # 第3步：按段落写入 Word
        for para in text.split('\n'):
            if para.strip():
                doc.add_paragraph(para.strip())
        if i < len(images) - 1:
            doc.add_page_break()
    
    doc.save(docx_path)
    return docx_path
OCR 前置条件：需用户自行安装 Tesseract OCR 引擎（含中文语言包）。安装程序会提示下载并自动配置路径。中文包：chi_sim.traineddata，约 30MB。

06 屏幕录制模块

技术路线选型

核心方案：pywin32 截图帧 → 内存缓冲 → FFmpeg 编码。这是最成熟、兼容性最好的 Windows 录屏方案，避免了 OBS 等重量级依赖。
方案优点缺点选用
pywin32 + FFmpeg轻量、兼容性好、可控性强CPU 占用稍高选用
Windows.Graphics.Capture APIGPU 加速、低 CPU 占用需 Win10 1903+，API 复杂备选
OBS Python 绑定功能强大依赖 OBS 安装，包体过大不选

录屏核心流程

import win32gui, win32ui, win32con
import subprocess, threading, queue
from PIL import Image
import numpy as np

class ScreenRecorder:
    def __init__(self, output_path: str, fps: int = 30, region: tuple = None):
        self.output_path = output_path
        self.fps = fps
        # region = (x, y, width, height)，None 表示全屏
        self.region = region or self._get_screen_size()
        self.running = False
        self.ffmpeg_proc = None
    
    def start(self, audio_source: str = "dshow"):
        self.running = True
        x, y, w, h = self.region
        
        # 启动 FFmpeg 进程，从 stdin 读取原始视频帧
        cmd = [
            "ffmpeg", "-y",
            "-f", "rawvideo", "-vcodec", "rawvideo",
            "-s", f"{w}x{h}", "-pix_fmt", "bgr24",
            "-r", str(self.fps), "-i", "-",  # 从 stdin 读帧
            "-f", "dshow", "-i", "audio=麦克风",  # 音频
            "-vcodec", "libx264", "-preset", "ultrafast",
            "-acodec", "aac", self.output_path
        ]
        self.ffmpeg_proc = subprocess.Popen(cmd, stdin=subprocess.PIPE)
        
        # 开启截图线程
        threading.Thread(target=self._capture_loop).start()
    
    def _capture_loop(self):
        """使用 GDI 高速截屏"""
        x, y, w, h = self.region
        hdesktop = win32gui.GetDesktopWindow()
        desktop_dc = win32gui.GetWindowDC(hdesktop)
        img_dc = win32ui.CreateDCFromHandle(desktop_dc)
        mem_dc = img_dc.CreateCompatibleDC()
        screenshot = win32ui.CreateBitmap()
        screenshot.CreateCompatibleBitmap(img_dc, w, h)
        mem_dc.SelectObject(screenshot)
        
        while self.running:
            mem_dc.BitBlt((0,0), (w,h), img_dc, (x,y), win32con.SRCCOPY)
            bmpinfo = screenshot.GetInfo()
            bmpstr = screenshot.GetBitmapBits(True)
            frame = np.frombuffer(bmpstr, dtype=np.uint8)
            self.ffmpeg_proc.stdin.write(frame.tobytes())
    
    def stop(self):
        self.running = False
        self.ffmpeg_proc.stdin.close()
        self.ffmpeg_proc.wait()

07 广告位系统设计

设计原则

广告系统与主功能完全解耦：主功能离线可用，广告网络请求在独立线程，失败时静默降级（不显示广告，不弹错误）。

广告内容由服务端通过 HTTP JSON API 下发，客户端定时轮询（默认每24小时一次），结果缓存到本地 SQLite：

客户端广告拉取逻辑

import requests, sqlite3, json
from datetime import datetime, timedelta
from PyQt6.QtCore import QThread, pyqtSignal

class AdFetcher(QThread):
    ad_ready = pyqtSignal(dict)   # 广告数据就绪信号
    
    def run(self):
        try:
            # 先检查本地缓存是否未过期
            cached = self._get_cache()
            if cached:
                self.ad_ready.emit(cached)
                return
            
            # 拉取最新广告（超时3秒，失败静默）
            resp = requests.get(
                self.api_url, params={"position": self.position},
                timeout=3, headers={"X-API-Key": self.api_key}
            )
            if resp.status_code == 200:
                data = resp.json()["data"]
                self._save_cache(data)
                self.ad_ready.emit(data)
        except:
            pass  # 失败静默：不显示广告，不影响主功能

10 数据存储设计

SQLite 数据库 Schema

数据库文件位于：%APPDATA%\BaibaoBOX\data.db

-- 操作历史记录表
CREATE TABLE history (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    module     TEXT    NOT NULL,  -- 'img' | 'pdf' | 'record'
    action     TEXT    NOT NULL,  -- 操作描述
    file_count INTEGER DEFAULT 0, -- 文件数量
    saved_mb   REAL    DEFAULT 0, -- 节省空间(MB)，图片压缩专用
    duration_s INTEGER DEFAULT 0, -- 时长(秒)，录屏专用
    output_dir TEXT,              -- 输出目录
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 广告缓存表
CREATE TABLE ad_cache (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    position   TEXT    UNIQUE NOT NULL, -- 广告位标识
    ad_data    TEXT    NOT NULL,        -- JSON 字符串
    expire_at  DATETIME,
    fetched_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

11 开发使用步骤（详细）

A. 开发环境搭建

1
安装 Python 3.11
前往 python.org 下载 Python 3.11.x（64位）安装包。
安装时务必勾选 "Add Python to PATH"，否则命令行无法识别。
安装完成后验证： 
python --version
# 输出：Python 3.11.x
2
克隆项目并创建虚拟环境
git clone https://github.com/your-org/baibaoxiang.git
cd baibaoxiang

# 创建独立虚拟环境（避免污染全局环境）
python -m venv venv

# 激活虚拟环境（Windows）
venv\Scripts\activate

# 激活后命令行前缀变为 (venv)
3
安装所有 Python 依赖
pip install -r requirements.txt

# requirements.txt 内容：
# PyQt6==6.6.1
# Pillow==10.3.0
# pdf2docx==0.5.8
# python-docx==1.1.0
# pdf2image==1.17.0
# pytesseract==0.3.10
# pywin32==306
# pyautogui==0.9.54
# requests==2.31.0
# PyInstaller==6.5.0
4
安装 FFmpeg（必须）
1. 前往 ffmpeg.org/download.html 下载 Windows 预编译版本（推荐 BtbN 版）
2. 解压后将 ffmpeg.exe 复制到项目 tools/ffmpeg/ 目录
3. 或添加到系统 PATH，使全局可用： 
setx PATH "%PATH%;C:\ffmpeg\bin"
5
（可选）安装 Tesseract OCR
仅在需要处理扫描版 PDF 时安装。
1. 下载：github.com/UB-Mannheim/tesseract/wiki（Windows 安装版）
2. 安装时勾选 "Chinese (Simplified)" 语言包
3. 安装路径记下来，后续在「系统设置」中填入： 
C:\Program Files\Tesseract-OCR\tesseract.exe
6
运行开发模式
# 确保虚拟环境已激活
cd src
python main.py

# 正常启动后会显示百宝箱主窗口
# 开发模式下修改代码后重启即可看到效果

B. 日常开发工作流

●
修改 UI 页面
编辑 src/ui/ 目录下的对应页面文件，UI 代码使用 PyQt6 组件树构建，样式通过 QSS 控制。
●
修改主题样式
编辑 themes/ 目录下的 .qss 文件，实时切换主题后立即生效，无需重启。
●
调试功能引擎
单独测试引擎模块时，可直接在命令行运行对应 Python 文件：
python src/engines/image_engine.py

12 打包发布流程

PyInstaller 打包配置

# build.spec - PyInstaller 配置文件
a = Analysis(
    ['src/main.py'],
    pathex=['.'],
    binaries=[
        ('tools/ffmpeg/ffmpeg.exe', 'tools/ffmpeg'),  # 打入FFmpeg
    ],
    datas=[
        ('themes/', 'themes'),     # 打入主题文件
        ('assets/', 'assets'),     # 打入图标资源
    ],
    hiddenimports=['PIL._tkinter_finder'],
)

pyz = PYZ(a.pure)
exe = EXE(
    pyz, a.scripts, a.binaries, a.zipfiles, a.datas,
    name='BaibaoBOX',
    icon='assets/icons/app.ico',
    console=False,         # 不显示控制台窗口
    uac_admin=False,       # 不需要管理员权限
)

一键构建命令

# 激活虚拟环境后执行
pyinstaller build.spec --clean --noconfirm

# 构建输出在 dist/BaibaoBOX/ 目录
# 预计大小：80~120MB（主要来自 FFmpeg）

广告位 & 系统功能
