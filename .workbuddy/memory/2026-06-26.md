## 启动崩溃修复（RuntimeError / 0xC0000005）

**问题**：`python main.py` 报 `RuntimeError: super-class __init__() of type SignalBus was never called`，后深挖发现是 PyQt6 在 `QApplication` 创建前不允许任何带 `pyqtSignal` 的 QObject 子类被实例化（甚至类定义也有时序限制）。

**根本原因**：`signals.py` 模块级直接执行 `bus = SignalBus()`，而 import 链（`main.py → main_window.py → signals.py`）发生在 `QApplication` 创建之前。

**修复方案**：
1. `signals.py` 改为懒加载单例：删除模块级 `bus = SignalBus()`，改为 `get_bus()` 函数 + 模块 `__getattr__` 兼容旧写法
2. `main.py` 在 `QApplication` 创建后立即调用 `get_bus()` 主动初始化
3. 验证：20个文件语法全通过，MainWindow 成功创建（1100x720），启动链完全正常

**文件变动**：`src/signals.py`（重写）、`main.py`（加 get_bus() 调用）
